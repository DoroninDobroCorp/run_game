## Текущий тезис

> **Твой город становится игровым уровнем: приложение строит тренировочный маршрут через реальные места вокруг дома и превращает их в сцены истории, которая могла произойти только здесь.**

Пользователь один раз задаёт стартовую точку, а перед занятием подтверждает тренировку текущего дня. Утверждённая beginner-программа определяет нагрузку. Детерминированная геосистема выбирает маршрут и POI. Автор задаёт мир и сцены. ИИ связывает эти слои, но не прокладывает путь, не придумывает факты о городе и не меняет физиологическую структуру тренировки.

MVP сначала проверяет именно связку `реальный маршрут → реальные места → локализованная история`, потому что обычные беговые таймеры, планы и GPS-трекинг уже многократно реализованы.

- [COMPETITIVE_RESEARCH.md](./COMPETITIVE_RESEARCH.md) — срез рынка до отдельного geo-narrative исследования;
- [MVP_DEVELOPMENT_BACKLOG.md](./MVP_DEVELOPMENT_BACKLOG.md) — прежняя последовательность шестимиссионного MVP без автоматической карты;
- [WORKING_NOTES.md](./WORKING_NOTES.md) — история идей и решений.

При любом противоречии действует порядок приоритета:

```text
GEO_NARRATIVE_PRODUCT_STRATEGY
  > GEO_NARRATIVE_TECHNICAL_SPEC
  > AI_FIRST_EXECUTION_PLAN
  > исторические документы
```

Исследование конкурентов сообщает факты и ограничения, но не переопределяет принятое продуктовое решение.
