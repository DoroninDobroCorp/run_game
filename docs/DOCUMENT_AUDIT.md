## 2. Противоречия между текущим видением и историческими документами

1. **Реалтайм vs Предварительная компиляция:**
   * *Устарело (COMMERCIAL_PRODUCT_PLAN):* ИИ пишет историю во время бега на основе пульса, темпа и голосового общения с игроком.
   * *Текущая стратегия:* Вся генерация происходит **до старта**. На пробежке работает локальный скомпилированный пакет (`RouteBundle`), чтобы избежать падения сети и задержек.
2. **Каталог маршрутов vs Единственная рекомендация:**
   * *Устарело:* Пользователь выбирает из каталога готовых маршрутов и POI.
   * *Текущая стратегия:* Принцип минимального выбора. Приложение выдает **одну рекомендуемую миссию** под текущий день тренировочного плана. Пользователь может лишь подтвердить или нажать «не подходит» (reroll).
3. **Роль ИИ в генерации канона:**
   * *Устарело:* Свободные разговоры с NPC и бесконечный сюжет от LLM.
   * *Текущая стратегия:* ИИ — это клей. LLM локализует готовый авторский шаблон под реальные места, но не выдумывает канон и не строит сам маршрут.

## 3. Решения основателя (Founder Decisions)

На основе обратной связи от основателя зафиксированы следующие решения:

1. **География продукта и исследования:**
   * **Решение:** Product behavior — **home-territory-based**: один пользователь развивает одну активную домашнюю территорию, а travel/cross-city continuity не являются фичами MVP. Founder research logistics — **traveler-based**: текущий город основателя используется как сменный полевой fixture без попадания города в канон или story state. Бар остаётся только frozen R01 fixture.
2. **Story-first R02 и стартовый канон:**
   * **Решение:** До универсального генератора обязателен минимальный трёхмиссионный authoring graph; production runtime graph engine откладывается. Выбрана оригинальная вселенная **«Нулевой слой»**, постоянный персонаж Леа и центральный поворот «маршруты игрока не спасали, а создавали её». Полностью производится пока только M1.
3. **Stop-loss (Ограничение потерь):**
   * **Решение:** Проект ведется в формате хобби с фокусом «сначала делаю для себя». Жестких временных рамок в месяцах нет. Финансовый лимит — использование персональных подписок на ИИ (до $200/мес) и бесплатных/дешевых тиров API провайдеров. Архитектура должна закладывать будущую монетизацию, но MVP собирается на минимальных костах.
4. **API-провайдеры (API Providers):**
   * **Решение:** Утверждены **GraphHopper** (с бесплатным лимитом 3000 запросов/день для тестов), **Overpass API** (бесплатный OSM) и **Wikipedia API** (бесплатный). Использовать персональные ключи основателя для LLM (OpenAI/Anthropic/Gemini) через файл `.env`.
5. **Безопасность маршрутов (Safety Review Owner):**
   * **Решение:** Двухуровневая система. Основатель вручную проверяет финальные маршруты перед выходом на тест. Дополнительно внедряется **автоматический ИИ-помощник (Smart AI Safety Validator)** как слой пре-валидации маршрутов и POI (отсечение шоссе, промзон, закрытых территорий по тегам и описаниям).

## 4. Аудит существующего кода основателя (TerraIncognita)

В ходе документального аудита был клонирован и проанализирован связанный проект основателя [TerraIncognita](https://github.com/DoroninDobroCorp/TerraIncognita). 

**Результаты анализа интеграции:**
* **Наличие:** В проекте `TerraIncognita` есть готовый FastAPI бэкенд и MapLibre фронтенд, которые содержат рабочие адаптеры, идеально подходящие для Run Game.
* **Что мы забираем/используем как референс:**
  * `backend/app/sources/osm.py` — готовый клиент для работы с Overpass API (запросы POI вокруг координат).
  * `backend/app/sources/wikidata.py` — парсинг метаданных и связей из Викиданных.
  * `backend/app/services/route_builder.py` и `route_optimizer.py` — логика построения маршрутов и обнаружения объектов вдоль коридора движения.
  * `backend/app/services/llm_storytelling.py` — базовая генерация нарративов на основе мест.
* **Вывод:** Существующий код основателя дает нам мощный фундамент для реализации `PoiProvider` и `RouteProvider`. Нам не нужно писать эти интеграции с нуля, мы адаптируем готовые решения из `TerraIncognita`.
